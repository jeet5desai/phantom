# Remove Organization Table — Scope Everything by `user_id`

## Summary

Replace `org_id` with `user_id` across the entire stack. The Organization model/table gets deleted. Every service, route, and the auth middleware will scope by `userId` (from the API key's `user_id` column) instead of `orgId`.

## Changes

### 1. Prisma Schema (`apps/api/prisma/schema.prisma`)

- **Delete** the `Organization` model entirely
- **Agent**: replace `orgId`/`org_id` → `userId`/`user_id`, drop the `organization` relation
- **Credential**: replace `orgId`/`org_id` → `userId`/`user_id`, drop the `organization` relation
- **ApiKey**: drop `orgId`/`org_id` column and the `organization` relation (already has `userId`)
- **AuditLog**: replace `orgId`/`org_id` → `userId`/`user_id`, rename index accordingly

### 2. Auth Middleware (`apps/api/src/middleware/auth.ts`)

- Remove `getOrganization` import and call
- Change `request.org` → `request.userId` (string, from `authenticateByApiKey`)
- Remove `request.apiKeyId` or keep it
- Remove the `Organization` type from Fastify request augmentation

### 3. Services

**Delete**: `organization.service.ts` — no longer needed

**`apikey.service.ts`**:

- Remove `ensureUserOrg`, `orgId` from all queries
- Remove `orgId` from `ApiKey` interface
- `authenticateByApiKey` returns `{ userId, keyId }` (no `orgId`)

**`agent.service.ts`**:

- Replace all `orgId` params → `userId`
- `CreateAgentInput.orgId` → `userId`
- All queries filter by `userId` instead of `orgId`

**`credential.service.ts`**:

- Replace all `orgId` params → `userId`

**`token.service.ts`**:

- `CreateTokenInput.orgId` → `userId`
- `revokeToken` / `revokeAllTokens`: filter via `agent.userId` instead of `agent.orgId`
- All `logAction` calls: `orgId` → `userId`

**`audit.service.ts`**:

- `LogActionInput.orgId` → `userId`
- All queries: `orgId` → `userId`
- Raw SQL in `logAction`: `org_id` → `user_id`

**`dashboard.service.ts`**:

- `getStats(orgId)` → `getStats(userId)`
- Remove `prisma.organization.findUnique` call for orgName
- All queries filter by `userId`

### 4. Routes

**Delete**: `organization.routes.ts` — no longer needed
**Remove** from `index.ts`: the `registerOrgRoutes` import and registration

**All other routes** (`agent`, `credential`, `token`, `permission`, `audit`, `dashboard`):

- Replace `request.org.id` → `request.userId`

### 5. Seed (`apps/api/src/db/seed.ts`)

- Remove `createOrganization` usage
- Use a hardcoded `userId` (e.g., `'seed-user'`) for all seed data

### 6. Dashboard Frontend

**`LayoutWrapper.tsx`**:

- Remove `orgName` fetching from dashboard stats (or replace with user name from Clerk)
- Show Clerk user info in sidebar instead of org name

**`page.tsx`** (dashboard):

- Remove `orgName` from `DashboardStats` interface (if present)

## Order of Operations

1. Schema + delete org service/routes
2. Update apikey service (remove orgId)
3. Update auth middleware (userId instead of org)
4. Update all services (agent, credential, token, audit, dashboard)
5. Update all routes
6. Update seed
7. Update dashboard frontend
